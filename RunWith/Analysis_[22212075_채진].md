## 1. Introduction

### 1) Summary

러닝은 특별한 장비 없이 언제 어디서든 할 수 있어서 꾸준히 하는 사람들이 많다. 그런데 혼자 뛰다 보면 금방 지루해지거나 페이스를 유지하기 힘든 경우가 많다. 같이 뛸 사람이 있으면 훨씬 더 오래 달릴 수 있다는 걸 알면서도, 막상 비슷한 페이스와 일정을 가진 러닝 파트너를 찾는 것이 쉽지 않다. 러닝 크루에 들어가자니 부담스럽고, 친구에게 부탁하자니 페이스가 안 맞는 경우도 많다.
그래서 러닝 파트너를 쉽게 찾게하기 위해서 RunWith를 만들려고 한다.

### 2) Business Goals

RunWith 시스템의 첫 번째 목표는 혼자 뛰는 사람들이 쉽게 러닝 파트너를 찾을 수 있도록 하는 것이다. 두 번째 목표는 페이스, 선호 거리, 시간대, 위치 등의 조건으로 나에게 맞는 파트너를 필터링할 수 있도록 하는 것이다. 세 번째 목표는 약속 잡기와 채팅 기능을 제공하여 매칭 이후의 과정도 앱 안에서 해결할 수 있도록 하는 것이다.

## 3. Domain analysis

RunWith 시스템의 주요 도메인 클래스는 아래와 같다.

| 클래스 | 속성 | 역할 |
|--------|------|------|
| User | userId, email, password, name | 시스템 사용자 |
| Profile | pace, preferDistance, preferTime, bio, rating | 러닝 프로필 정보 |
| Matching | matchId, requesterId, receiverId, status | 매칭 요청 및 상태 관리 |
| ChatRoom | roomId, userId1, userId2, createdAt | 채팅방 정보 |
| Message | messageId, roomId, senderId, content, sentAt | 채팅 메시지 |
| Appointment | appointmentId, matchId, date, time, location | 러닝 약속 정보 |
| Review | reviewId, appointmentId, rating, comment | 리뷰 및 평점 |

## 5. Glossary

| 용어 | 설명 |
|------|------|
| 매칭 (Matching) | 두 사용자가 서로 러닝 파트너로 연결되는 것 |
| 페이스 (Pace) | 1km를 달리는 데 걸리는 시간. 예: 6분/km |
| 신뢰도 점수 | 리뷰와 약속 이행 여부를 바탕으로 계산되는 사용자의 신뢰 지표 |
| 노쇼 (No-show) | 러닝 약속을 잡고 나타나지 않는 행위 |
| 푸시 알림 | 약속 당일, 매칭 요청 등 주요 이벤트 발생 시 앱이 보내는 알림 |
| 차단 | 특정 사용자와 더 이상 매칭되지 않도록 설정하는 기능 |
| Use Case | 사용자가 시스템과 상호작용하는 기능 단위 |
| Actor | 시스템과 상호작용하는 외부 주체 |
